# Standalone AI API Backend

This is the standalone Python FastAPI backend for the AI Chat application. While the root repository contains a full-stack deployment (Frontend + Backend), this directory contains the configuration needed to deploy and run **only** the API. This acts as a headless backend for any external frontend application you wish to build.

## 🚀 Features

- **Headless API Mode**: Automatically skips mounting static frontend files when deployed via the included `docker-compose.yml` (`API_ONLY=true`).
- **CORS Enabled**: Configured to accept cross-origin requests, allowing frontends hosted on completely different domains (like Vercel, Netlify, or a local dev server) to communicate seamlessly.
- **FastAPI Framework**: High-performance, asynchronous routing.
- **Local LLM Integration**: Uses the official `ollama` Python library to stream real-time responses from a local Qwen 2.5 model.
- **JWT Authentication**: Protects the AI endpoints with a simple, database-free token validation layer.

## 🔒 Authentication Details

The API requires a Bearer token for accessing the `/api/chat` endpoint. You can obtain this token by sending a `POST` request to `/api/login` with the following default credentials:
- **Username:** `admin`
- **Password:** `password123`

*(You can modify these hardcoded credentials directly inside `main.py`)*.

## 🐳 Docker Deployment

You can deploy this API entirely independently using the provided Docker configuration inside this folder. It utilizes `python:3.12-slim` and connects to an existing Docker network to communicate with Ollama.

### 1. Configure the Network
Ensure you have created the shared Docker network that your Ollama container is already connected to:
```bash
docker network create ai-network
```

### 2. Deploy the API
Run the following command from **inside** the `backend/` directory:
```bash
docker compose up --build -d
```

The API will now be instantly accessible at `http://localhost:8000`.

## 📡 API Endpoints

### `POST /api/login`
Validates user credentials and returns an access token.

**Request Body:**
```json
{
  "username": "admin",
  "password": "password123"
}
```
**Response:**
```json
{
  "access_token": "secret-token-12345",
  "token_type": "bearer"
}
```

### `POST /api/chat`
Streams a response from the LLM based on the full conversation history.

**Headers:**
- `Authorization: Bearer <your-access-token>`
- `Content-Type: application/json`

**Request Body:**
```json
{
  "messages": [
    {
      "role": "user",
      "content": "Hello! Who are you?"
    }
  ]
}
```
**Response:** 
Returns a `text/plain` stream of characters generated by the AI in real time.

## 💻 Local Development

If you wish to test or modify the API locally without Docker:

```bash
# Create a virtual environment using uv
uv venv
# Install dependencies from pyproject.toml
uv pip install -e .
# Run the FastAPI server with hot-reloading
uv run uvicorn main:app --reload
```
